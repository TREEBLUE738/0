On-Request Languages Management

Process Overview

Process initiated at least 8 weeks before PCY starts. PCY defines on-request language choices and estimated number of meetings for all Council Preparatory Bodies (CPBs) under on-request regime.

Step Actor Action Details
1 Presidency Send on-request choices Define priority languages for each CPB, validity period, and estimated number of meetings
2 SCIC Officer Acknowledge choices Updates internal records in PEARL system
3 Meeting Planner Adjust if needed When Member State requests modification
4 SCIC Officer Acknowledge updates Confirms receipt of changes

Creation of New Meeting Group

Process Overview

Separate workflow for creating new meeting groups with unique SCIC codes. Not part of the main timeline but essential for system setup.

Step Actor Action Required Information
1 Meeting Planner Send new meeting group to SCIC Title (EN/FR), active/passive languages, priority languages, unique SCIC code
2 SCIC Officer Create meeting group in PEARL Updates meeting group database with all details

Interpretation Order Types Reference

Order Type Purpose Who Creates Trigger SCIC Role Cost Impact Key Rules / Notes
New Interpretation Order Request interpretation for a new session/meeting Council (Meeting Planner via POMME) Planned (8-weeks cycle) or ad-hoc/urgent request SCIC (PEARL) checks feasibility → Accept / Reject / Expire. Rejection justification is OPTIONAL None (normal order flow) If urgent, Council sets deadline for SCIC reply. Accepted order requires SCIC to send Provided Regime
Cancellation Order (aka Cancellation request) Cancel a session + interpretation teams Council (POMME) Meeting/session no longer takes place SCIC acknowledges receipt and updates status Before SLA → no cost
After SLA → cancellation with cost 		If delay is foreseen and justification accepted, cancellation may still be without cost
Replacement Order (aka Reallocation of Standby Teams) Reassign interpreters from cancelled session to another Council (POMME) Session cancelled but interpreters can still be reused SCIC checks compatibility (same day, time slot, number of interpreters) Avoids cancellation costs if compatible If leftover languages: before SLA → no cost, after SLA → with cost
Language Regime Change Add/remove languages to accepted order Council (POMME) Delegation needs change SCIC checks feasibility of added languages Add languages → penalty-free anytime
Remove before SLA → no cost
Remove after SLA → with cost 		Updates existing order. All changes tracked against same order ID
